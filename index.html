HTML

  • HTML is used as the building block or foundation for web applications. It is like the frame of a house.
  • Header: Information in the header is not visible to the viewer of the webpage (except for the title). The information in the header is to instruct the browser how to render the page.
  • Body: Anything that should appear on the webpage needs to be in between the opening and closing tag of the Body.
  • Sections: These are containers that help to separate different kinds of content on a page. You can have sections within sections.
  • Every HTML element is essentially a box that has margins outside of the element (box) and padding inside the element around the conent inside contents.

CSS

  • CSS is used to add styling to an application.
  • Don't Repeat Yourself (DRY) code helps keep codebases as compact as possible - reducing size and confusion.
  • Using an HTML element's name as a selector and adding an attribute in the CSS file will apply those properties to all elements with that name.
  • Using a class selector and attribute allows us to be selective about which elements have certain properties.
  • Add the .class attribute to the CSS file first - then add the selector to the corresponding HTML element
  • The class attribute defines what properties we want elements in the class to have.
  • The class selector is what is added to the HTML elements that you want to have those attributes defined in the CSS file.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it

JavaScript

  • JS is what adds interactive capabilities to an application.
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Functions need 2 parts to process correctly: define the function and call the funcation.
